# PostgreSQL 部署与迁移指南

AList-TvBox 支持三种主应用数据库：**H2**（内置默认）、**MySQL**、**PostgreSQL**。本文说明如何新建 PostgreSQL 部署，以及如何把现有 H2 / MySQL 数据迁移到 PostgreSQL。

> 内嵌 AList 运行时数据库是独立的一套（见文末），与主应用库互不影响。

---

## 一、新建 PostgreSQL 部署

### 方式 A：管理脚本（推荐）

```bash
./alist-tvbox.sh config-db
# 选择 3) PostgreSQL，填入 JDBC URL / 用户名 / 密码
./alist-tvbox.sh config-db apply   # 用新配置重建容器
```

脚本会把数据库配置写入 `~/.config/alist-tvbox/app.conf`，并在重建容器时注入以下环境变量：

- `SPRING_DATASOURCE_JDBC-URL`
- `SPRING_DATASOURCE_USERNAME` / `SPRING_DATASOURCE_PASSWORD`
- `SPRING_DATASOURCE_DRIVER-CLASS-NAME` = `org.postgresql.Driver`
- `SPRING_JPA_DATABASE-PLATFORM` = `org.hibernate.dialect.PostgreSQLDialect`
- `SPRING_PROFILES_ACTIVE` = `postgresql`（叠加在镜像默认 profile 之上）

### 方式 B：直接用环境变量

无需脚本，自行注入上述环境变量即可。首次启动 Flyway 会自动建表（`V1`–`V5`），Hibernate `ddl-auto=validate` 校验 schema 与实体一致。

```bash
docker run -d ... \
  -e SPRING_PROFILES_ACTIVE=postgresql \
  -e SPRING_DATASOURCE_JDBC-URL='jdbc:postgresql://<host>:5432/alist_tvbox' \
  -e SPRING_DATASOURCE_USERNAME=atv \
  -e SPRING_DATASOURCE_PASSWORD='<密码>' \
  -e SPRING_DATASOURCE_DRIVER-CLASS-NAME=org.postgresql.Driver \
  -e SPRING_JPA_DATABASE-PLATFORM=org.hibernate.dialect.PostgreSQLDialect \
  <镜像>
```

切回 H2：运行 `config-db` 选择 1，再 `config-db apply`（清空外部数据库配置，走镜像内置 H2）。

## 二、从 H2 / MySQL 迁移到 PostgreSQL

迁移基于**跨数据库的 JSON 备份/恢复**（走 JPA，与具体数据库无关），分两步：

### 第 1 步：在源实例（H2 / MySQL）导出

```bash
./alist-tvbox.sh migrate-db export
# 读取 <数据目录>/backup_token，调用 POST /api/local/backup?format=json
# 生成当前目录下的 database-json.zip
```

> 若没有 `backup_token`，先通过管理界面或重置流程生成（同本地备份的鉴权方式）。

### 第 2 步：在目标 PostgreSQL 实例导入

先确保目标实例已按「一、新建部署」指向一个**空的** PostgreSQL 库（首次启动 Flyway 建表）。然后：

```bash
./alist-tvbox.sh migrate-db import database-json.zip
# 复制到 <数据目录>/database-json.zip 并重启容器
```

容器重启后，`StartupJsonRestoreRunner` 会读取该 zip 恢复数据，恢复完成后自动重启（exit code 85）以清缓存重新初始化。观察日志：

```bash
docker logs -f <容器名>
```

---

## 三、注意事项

- **IDENTITY 表的 id**：`account` / `session` / `user` 三张表使用数据库自增列。导入到空库时，恢复逻辑按业务键 upsert；管理员账号会被 `ensureAdminOccupiesIdOne()` 重新固定到 `id=1`，保证启动初始化不会重复创建 admin。
- **只迁移主应用库**：`migrate-db` 仅迁移主 tvbox 应用库。内嵌 AList 运行时库需单独处理（见下）。
- **JSON 路径可移植，SQL 路径不可移植**：H2 的 `SCRIPT` SQL 备份只能用于 H2；迁移到 PostgreSQL 必须用 JSON 备份（`format=json`）。`migrate-db` 已默认走 JSON。
- **Schema 一致性**：PostgreSQL 的 `V1` 与 H2/MySQL 的 `V1` 列名、表名完全一致，仅类型/语法按方言调整（`AUTO_INCREMENT`→`GENERATED BY DEFAULT AS IDENTITY`、`TINYINT`→`SMALLINT`、`LONGTEXT`→`TEXT`、保留字改用双引号）。

## 四、内嵌 AList 运行时数据库

内嵌 AList 的数据库由其 `config.json` 的 `database.type` 决定，支持：

- `sqlite3`（默认）
- `mysql`
- `postgres`

切换为 PostgreSQL 时，编辑 AList 的 `config.json`：

```json
{
  "database": {
    "type": "postgres",
    "host": "localhost",
    "port": 5432,
    "user": "atv",
    "password": "<密码>",
    "name": "alist",
    "ssl_mode": "disable"
  }
}
```

应用读取到 `type: "postgres"` 后，会用 `org.postgresql.Driver` 连接对应的库。
